Joker is currently in development. It's on the fast track to a 1.0 release, so we encourage you to use it and give us your feedback, but there are things that haven't been finalized yet and you can expect some changes.

An modern and intuitive testing for command-line apps.

Installation

$ npm install @amilajack/joker

How it looks

import Joker from '@amilajack/joker';

new Joker()
  .exec('touch /tmp/test')
  .run('ls /tmp/')
  .stdout(/test/)
  .end();

Formatting options

joker can strip newlines and colors. You can tell it to do so by passing an object that looks like this:

const options = {
  colors: false,
  newLines: false,
};

Joker(options).stdout...

Custom expectations

While joker comes with built-in expectations, you can use your own too.

new Joker()
  .expect((result) => {
    if (result.stdout !== 'unicorns') {
      return new Error('NO!');
    }
  })
  .run('unicorns')
  .end(fn);

Custom middlewares

You can register as many before and after middlewares as you wish.

new Joker()
  .before(setupDatabase)
  .before(runMigrations)
  .run(cmd)
  .after(downgradeCron)
  .after(deleteDatabase)
  .end();

Middleware order

The Middleware execution order is very simple - "before" middlewares always run before everything else, "after" middlewares always run after everything else. The other middlewares will match the order that you have specified.

new Joker()
  .before(before1)
  .before(before2)
  .after(after1)
  .after(after2)
  .writeFile(file, '')
  .run(cmd)
  .unlink(file)
  .end(fn);

// Execution order:
// before1, before2, writeFile, cmd, unlink, after1, after2

You may also want to reuse before and after middlewares as much as possible, especially when testing something that requires extensive setup and cleanup. You can accomplish this by cloning a joker instance.

const base = new Joker()
  .before(setupDatabase)
  .after(removeDatabase);

// Later on

base.clone().run....

Plugins

joker has primitive support for plugins. You can register any expectation or/and any middleware by calling joker.register.

const fn = () => {};
new Joker().register('foo', fn);

Or you may want to register many functions at once.

const fn = () => {};
const fn1 = () => {};
joker.register({ baz: fn, bar: fn1 });

Usage with a test runner

joker plays nice with any test runner out there. Here is a minimal example how you could use it with Mocha.

describe('todo add', () => {
  it('adds a new todo item', (done) => {
    new Joker()
      .run('todo add')
      .stdout('A new todo has been added')
      .end(done);
  });
});

Usage without a test runner

While using a test runner is recommended joker is completely 'nodeable'. Here is a simple example how you could accomplish that:

const assert = require('assert');

function refute(err) {
  assert(!err);
}

new Joker()
  .run(cmd)
  .end(refute);

new Joker()
  .run(anotherCmd)
  .end(refute);

Responding to interactive prompts

joker can respond to apps that run interactively using the on() and respond() functions.

new Joker()
  .run(cmd)
  .on('Your name: ')
  .respond('Joe User\n')
  .end();

See test/prompt.test.js for more examples.

API

#before

Register a "before" middleware.

new Joker()
  .before(fn)
  .before(fn2)
  .run(cmd)
  .end();

#after

Register an "after" middleware.

new Joker()
  .run(cmd)
  .after(fn)
  .after(fn2)
  .end();

#cwd

Change the current working directory of the main command (specified with run). Please note that this won't affect any other commands like unlink etc.

new Joker()
  .cwd(__dirname)
  .run('pwd')
  .stdout(/test$/)
  .end();

#base

Set a base command. Useful for templates.

new Joker()
  .base('node ')
  .run('--version')
  .stdout('0.10.16')
  .end();

#run

Set a primary command to execute:

new Joker()
  .run('node --version')
  .stdout('0.10.16')
  .end(fn);

You could also run the test right after specifying the command to run:

new Joker()
  .stdout('0.10.16')
  .run('node --version', fn);

#stdin

Set the contents of stdin.

new Joker()
  .stdin('foobar')
  .run('rev')
  .stdout('raboof')
  .end(fn);

#env

Set environment variables.

new Joker()
  .env('foo', 'bar')
  .env('baz', 'boo')
  .run('node --version')
  .stdout('0.10.16')
  .end(fn);

#timeout

Set a timeout for the main command that you are about to test.

new Joker()
  .timeout(1) // ms
  .run('cat /dev/null')
  .end(fn);

#stdout

Set expectations on stdout.

new Joker()
  .stdout('LICENSE Makefile')
  .run('ls')
  .end(fn);

Works with regular expressions too.

new Joker()
  .stdout(/system/)
  .run('time')
  .end(fn);

#stderr

Same as stdout but well.. surprise works with stderr.

new Joker()
  .run('todo add')
  .stderr('Please speicfy a todo')
  .end(fn);

#code

Expect a given exit code.

new Joker()
  .run('todo add')
  .code(1)
  .end(fn);

#exist

Check if a given path exists (works with both files and directories).

new Joker()
  .run('mkdir /tmp/test')
  .exist('/tmp/test')
  .end(fn);

#match

Check the contents of a file.

new Joker()
  .writeFile(file, 'Hello')
  .run('node void.js')
  .match(file, 'Hello')
  .unlink(file)
  .end(done);
new Joker()
  .writeFile(file, 'Hello')
  .run('node void.js')
  .match(file, /ello/)
  .unlink(file)
  .end(done);

#mkdir

Create a new directory.

new Joker()
  .mkdir('xml-database')
  .run('this does stuff with the xml-database directory')
  .end(fn);

#exec

Execute a given command.

new Joker()
  .writeFile('LICENSE', 'MIT License')
  .exec('git add -a')
  .exec('git commit -m "Add LICENSE"')
  .run('git log')
  .stdout(/LICENSE/)
  .end();

By default the commands will inherit the "world" for the main command which includes environment variables, cwd, timeout. However, you can override this by supplying a different "world":

new Joker()
  .exec('git add LICENSE', { timeout: 4, cwd: '/tmp' })
  .run('git log')
  .stdout(/LICENSE/)
  .end();

#writeFile

Create a file with or without given contents.

Without:

new Joker()
  .writeFile(pathToFile)
  .end();

With:

new Joker()
  .writeFile(pathToFile, data)
  .end();

#rmdir

Remove a directory.

new Joker()
  .mkdir('xml-database')
  .run('this does stuff with the xml-database directory')
  .rmdir('xml-database')
  .end(fn);

#unlink

Unlink a file.

new Joker()
  .writeFile('my-file', data)
  .run('this does stuff with my file')
  .unlink('my-file')
  .end(fn);

#on

Detect a prompt for user input. Accepts a String or RegExp that appears in the the stdout stream. Must be paired with #respond.

new Joker()
  .run(cmd)
  .on('Your name: ')
  .respond('Joe User\n')
  .end();

#respond

Write a response to the stdin stream when a prompt is detected.

See #on

#end

Run the given test.

new Joker()
  .run('ls')
  .stdout('this-is-not-porn-i-promise')
  .end((err) => {});

The same might be accomplished with supplying a function to run:

new Joker()
  .stdout('this-is-not-porn-i-promise')
  .run('ls', (err) => {});

#clone

Deep clone a joker instance.

const clone = new Joker()
  .before(fn)
  .after(fn)
  .run('my awesome command')
  .end()
  .clone();

#expect

Register a custom expectation.

new Joker()
  .expect((result) => {
    if (result.stdout !== 'Unicorns') {
      return new Error('OMG');
    }
  })
  .run('ls')
  .end(fn);
